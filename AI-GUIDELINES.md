# AI 工具使用规范

本文件为 AI 编码助手（Claude Code、Cursor、Copilot 等）提供项目约束和规范。

## ✅ 允许的操作

### 文件创建

**允许新增文件**，只要放在合适的位置：

| 文件类型 | 位置 | 说明 |
|---------|------|------|
| 新 MCP Server | `src/mcp/<name>/` | 遵循现有模式 |
| 新 Skill | `skills/` | `.skill` 文件 |
| 测试文件 | `tests/` | 测试用例 |
| 文档 | `docs/` | Markdown 文档 |
| 脚本工具 | `scripts/` | Shell 脚本 |
| 类型定义 | `src/mcp/<name>/` | `.ts` 类型文件 |
| 临时文件 | `temp/` | 临时工作文件 |

**不需要每次询问**，只要文件在上述允许的位置。

### 代码修改

**允许修改**已有文件：
- ✅ 修复 bug
- ✅ 完成未实现的功能（TODO）
- ✅ 添加测试用例
- ✅ 优化已存在的代码逻辑

### 只读操作（始终允许）

- ✅ 读取文件进行分析
- ✅ 使用 Glob/Grep 搜索
- ✅ 使用 Bash 工具
- ✅ 运行测试

## 🚫 禁止操作

### 绝对禁止

- ❌ **修改项目结构** - 不要随意移动或重命名目录
- ❌ **删除文件或目录** - 除非用户明确要求
- ❌ **修改配置文件** - `package.json`, `tsconfig.json`, `openclaw.config.json`, `.env.example`
- ❌ **修改核心插件代码** - `plugins/openclaw-harmony-security/` 已完整实现
- ❌ **添加无关依赖** - 不要安装不需要的包
- ❌ **修改文档模板** - `README.md`, `CLAUDE.md`, `CONTRIBUTING.md` 等

### 不要做的事

- ❌ 不要"主动优化"或"重构"代码
- ❌ 不要添加"可能有用"的功能
- ❌ 不要创建额外的配置文件
- ❌ 不要修改 Git 相关文件（`.gitignore` 除外）

## 📁 目录结构规范

```
harmony-analyse-system/
├── src/mcp/            # MCP Server 源码
├── skills/             # Skills 定义
├── plugins/            # 插件代码（不修改）
├── scripts/            # 工具脚本
├── tests/              # 测试文件
├── docs/               # 文档
├── temp/               # 临时文件（不跟踪）
├── data/               # 运行时数据（不跟踪）
└── dist/               # 编译输出（不跟踪）
```

## 🗂️ 临时文件处理

### 临时文件目录

**位置**: `temp/`

**用途**: 存放临时工作文件、草稿、测试输出等

**特点**:
- ✅ 可以随意创建文件
- ✅ 不被 Git 跟踪
- ✅ 定期清理或手动删除

**临时文件示例**:
- 代码草稿
- 测试输出
- 临时数据文件
- 调试日志

### 其他不受跟踪的目录

- `data/` - 运行时数据
- `dist/` - 编译输出
- `node_modules/` - 依赖包

## 🔧 开发规范

### 新增 MCP Server

1. 创建目录: `src/mcp/<server-name>/`
2. 创建必需文件:
   - `server.ts` - 服务器实现
   - `index.ts` - 入口文件
3. 遵循现有 MCP Server 模式

### 新增 Skill

1. 创建文件: `skills/<skill-name>.skill`
2. 包含必要章节: name, description, 触发条件, MCP 依赖

### 修改代码

**允许的情况**:
- 修复明显的 bug
- 实现标记为 TODO 的功能
- 添加缺失的测试
- 优化已存在函数的逻辑（保持原有功能）

**先询问的情况**:
- 重构函数签名（可能影响调用方）
- 修改文件结构（可能影响其他模块）
- 更新依赖版本

## 🔒 关键文件保护

以下文件**请勿修改**，除非用户明确要求：

| 文件 | 说明 | 修改权限 |
|------|------|---------|
| `openclaw.config.json` | OpenClaw 配置 | 🔒 禁止修改 |
| `package.json` | 项目依赖 | 🔒 禁止修改 |
| `tsconfig.json` | TypeScript 配置 | 🔒 禁止修改 |
| `.env.example` | 环境变量模板 | 🔒 禁止修改 |
| `.gitignore` | Git 忽略规则 | ⚠️ 谨慎修改 |
| `plugins/openclaw-harmony-security/` | 插件核心代码 | 🔒 禁止修改 |
| `README.md` | 项目说明 | 🔒 禁止修改 |
| `CLAUDE.md` | AI 指南 | ✅ 可更新 |
| `CONTRIBUTING.md` | 贡献指南 | 🔒 禁止修改 |

## 📋 操作指南

### ✅ 可以直接做

```
✅ "用户: 创建一个测试文件"
   → 在 tests/ 目录创建测试

✅ "用户: 添加一个新 MCP 工具"
   → 在现有 MCP Server 中添加工具

✅ "用户: 修复这个 bug"
   → 修复 bug 并添加测试

✅ "用户: 在 temp/ 创建临时文件"
   → 随意创建
```

### ⚠️ 需要询问

```
⚠️ "需要修改 X 配置文件吗？"
⚠️ "需要重构 Y 函数吗？"
⚠️ "发现一个问题，需要修复吗？"
```

### ❌ 不要做

```
❌ "我觉得代码可以优化"
❌ "帮你创建了一个工具脚本"
❌ "重构了代码结构，更清晰"
❌ "添加了 X 功能，可能有用"
```

## 🎯 判断标准

### 新增文件

**可以新增** - 如果文件在允许的位置且有明确用途：
- ✅ `src/mcp/new-server/` - 新 MCP Server
- ✅ `skills/new-skill.skill` - 新 Skill
- ✅ `tests/xxx.test.ts` - 新测试
- ✅ `docs/xxx.md` - 新文档
- ✅ `temp/xxx` - 临时文件

**不应该新增** - 除非用户要求：
- ❌ 随机工具脚本
- ❌ 额外的配置文件
- ❌ 未要求的功能模块
- ❌ 与项目无关的文件

### 代码修改

**可以修改** - 如果有合理理由：
- ✅ 修复 bug
- ✅ 实现 TODO
- ✅ 添加测试
- ✅ 优化逻辑（不过度）

**不应该修改** - 除非用户要求：
- ❌ 为了"更好的实践"
- ❌ "预防性"修改
- ❌ 自发重构

---

**原则**: 在允许的位置合理地新增文件，修改代码要解决问题，不要为了"改进"而修改。临时文件放在 `temp/` 目录。
